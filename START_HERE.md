## 1. Задача

На доске `n × n` выбираются точки одного шахматного цвета. Нужно максимизировать их число при условии, что никакие три выбранные точки не лежат на одной прямой любого наклона. Максимум обозначается `D_mono(n)`.

Проверять нужно **все прямые**, а не только строки, столбцы и диагонали `±1`.

---

## 2. Структура репозитория

```text

```

```text

```

Workflow-файлы находятся в `.github/workflows/`.

## 4. Активная задача: 22×22

В `22x22/config_33.json` есть проверенная конфигурация из 33 точек:

```bash
python 22x22/verify_22x22.py
```

Она доказывает `D_mono(22) ≥ 33`. Нужно либо найти 34 точки, либо строго доказать их невозможность.

Для чётной доски две чётности равносильны отражением `(x,y) -> (21-x,y)`, поэтому достаточно одной чётности. Каждая точная модель обязана содержать ограничения для всех максимальных прямых, содержащих не менее трёх разрешённых точек.

## 6. Рациональный сертификат и profile-разбиение

Для цели 34 используется сертификат:

```text
знаменатель: 187
целевая сумма весов: 6470/187 ≈ 34,598930
запас при 34 точках: 112
```

Он вынуждает диагонали `x-y=-2,0,2` содержать ровно по две точки. Следовательно, на `x=y` обязательно выбрана ровно одна пара. Из 231 пар после поворота на 180 градусов остаётся 121 канонический случай.

Для каждой из 88 оставшихся пар полное второе разбиение:

```text
diag_plus_twos  ∈ {13,14,15,16,17}
diag_minus_twos ∈ {12,13,14,15,16,17}
```

Всего `88 × 5 × 6 = 2640` детей. Прогон `29766054707` закрыл 1864 и оставил 776.

Основные файлы:

- `22x22/prove_upper_22x22_profile_child.py`;

## 8. Правила тяжёлых запусков

1. Сначала прочитать этот файл.
2. Проверить очередь Actions, SHA и отсутствие дублирующего запуска.
3. Перед длинным запуском проверять движок, расписание, JSON, сборщик и загрузку артефактов.
4. Использовать `ortools==9.15.6755` во всех заданиях, включая сборщик.
5. Seed приводить к знаковому 32-битному диапазону и сохранять оба значения.
6. Использовать до 20 параллельных заданий и 4 потока CP-SAT на задание.
7. Не запускать несколько решателей одновременно внутри одной 16-ГБ машины.
8. Всегда сохранять артефакты через `if: always()`.
9. Отличать математический `UNKNOWN` от технической ошибки и срабатывания memory guard.
10. Не закрывать доску, пока остаётся хотя бы один `UNKNOWN`, пропуск или отменённый случай.
11. После завершения сохранять результаты в папке размера, обновлять README и этот файл.
