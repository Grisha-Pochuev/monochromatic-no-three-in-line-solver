## Задача

На доске `n × n` выбираются точки одного шахматного цвета. Никакие три выбранные точки не должны лежать на одной прямой любого наклона. Максимум обозначается `D_mono(n)`.

## Структура репозитория

```text

...
```

Каждый значимый запуск хранится внутри папки своей доски:

```text

```

## Критический аудит модели 22×22

Полный разбор находится в:

```text
22x22/SYMMETRY_AUDIT_2026-07-22.md
```

В `prove_upper_22x22_profile_child.py` одновременно применялись два правила ориентации относительно транспонирования:

```text
row_twos <= column_twos
above <=lex below
```

Их безусловное сочетание некорректно: при `row_twos < column_twos` одно правило может выбрать исходную конфигурацию, а другое — транспонированную, после чего обе удаляются.

Безопасная точка входа:

```text
22x22/prove_upper_22x22_profile_child_safe.py
```

Она сохраняет все точные ограничения, но отключает безусловный lex-break и оставляет только:

```text
row_twos <= column_twos
```

Старый файл хранится только для воспроизводимости прежних запусков. Новые доказательные проверки должны использовать безопасный файл.

## Прогон 29918490062

Он разделил ребёнка `652` на 21 пару:

```text

```

Четыре объявленных `INFEASIBLE`:

```text

17 = (15,17)
19 = (16,17)
```

Все четыре имеют `row_twos < column_twos` и затронуты ошибкой. Их нужно считать открытыми. Для ребёнка `652` сейчас открыты **все 21 подслучай**.

Независимая локальная MILP-проверка HiGHS без lex-break дала для этих четырёх случаев только `UNKNOWN`.

## Сертифицированный статус

Ошибка не затрагивает проверенную нижнюю конструкцию, поэтому общая граница остаётся:

```text
33 <= D_mono(22) <= 34
```
