## Задача

На доске `n × n` выбираются точки одного шахматного цвета. Никакие три выбранные точки не должны лежать на одной прямой любого наклона. Максимум обозначается `D_mono(n)`.

## Структура репозитория

Каждый размер имеет отдельную папку в корне:

```text

18x18/
19x19/
20x20/
21x21/
22x22/
23x23/
...
```

Каждый значимый запуск хранится внутри папки своей доски:

```text
NxN/runs/YYYY-MM-DD-run-RUN_ID/
```

Локальные исследования веб-чата, которые не являются GitHub Actions runs, хранятся внутри `NxN/webchat/`.

## Критический аудит модели 22×22

Полный разбор:

```text
22x22/SYMMETRY_AUDIT_2026-07-22.md
```

Историческая модель `prove_upper_22x22_profile_child.py` одновременно применяла:

```text
row_twos <= column_twos
above <=lex below
```

Их безусловное сочетание некорректно: два правила могут выбрать противоположные ориентации и удалить обе транспонированные конфигурации.

Безопасная точка входа:

```text
22x22/prove_upper_22x22_profile_child_safe.py
```

Она отключает безусловный lex-break и оставляет только безопасное правило:

```text
row_twos <= column_twos
```

Материалы:


Параметр `A` считает выбранные точки на пересечениях:

```text

```

Точный рациональный двойственный сертификат доказывает:

```text
A <= 803/273 < 3
```

Следовательно, для целочисленной конфигурации:

```text
A <= 2
```

Проверка:

```bash
cd 22x22/webchat/2026-07-23-certificate-lp-triage
```

Этот результат строгий, использует точную арифметику и не зависит от ошибочного lex-break.

## Правила тяжёлых запусков

1. Сначала прочитать этот файл и аудит симметрии.
2. Не использовать старую profile-child модель как доказательную.
3. Использовать `prove_upper_22x22_profile_child_safe.py`.
4. Использовать `ortools==9.15.6755` для воспроизводимых CP-SAT проверок.
5. `UNKNOWN` не является доказательством.
6. Численное LP `INFEASIBLE` считать доказательством только после рационального двойственного сертификата.
7. Не пересчитывать 1510 аналитически исключённых детей.
8. Не запускать GitHub Actions без прямого указания пользователя.
9. После значимого исследования сохранять проверяемые материалы в папке соответствующего размера.
